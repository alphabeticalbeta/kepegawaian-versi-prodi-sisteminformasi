Test Admin Fakultas Workflow - Fixed

Test Case 1: Status "Diajukan"

Kondisi:

  • • Status: "Diajukan"
  • • Role: Admin Fakultas
  • • Usulan baru dari pegawai

Hasil yang Diharapkan:

  • ✅ Dokumen Admin Fakultas TIDAK ditampilkan
  • ✅ Tombol "Perbaikan Usulan" muncul
  • ✅ Tombol "Usulkan ke Universitas" muncul
  • ✅ Form validasi bisa diedit

Test Case 2: Status "Perbaikan Usulan"

Kondisi:

  • • Status: "Perbaikan Usulan"
  • • Role: Admin Fakultas
  • • Usulan dikembalikan dari Admin Univ Usulan

Hasil yang Diharapkan:

  • ✅ Dokumen Admin Fakultas DITAMPILKAN
  • ✅ Form dokumen BISA DEDIT
  • ✅ Tombol "Kirim Kembali ke Universitas" muncul
  • ✅ Form validasi bisa diedit

Test Case 3: Status "Diusulkan ke Universitas"

Kondisi:

  • • Status: "Diusulkan ke Universitas"
  • • Role: Admin Fakultas
  • • Usulan sudah divalidasi dan dikirim

Hasil yang Diharapkan:

  • ✅ Dokumen Admin Fakultas DITAMPILKAN
  • ❌ Form dokumen TIDAK BISA DEDIT
  • ❌ Tidak ada tombol action
  • ❌ Form validasi TIDAK BISA DEDIT

Test Case 4: Alur Return

Kondisi:

  • • Admin Univ Usulan meminta perbaikan
  • • Bisa return ke Admin Fakultas
  • • Bisa return ke Pegawai

Hasil yang Diharapkan:

  • ✅ Admin Univ Usulan bisa return ke Admin Fakultas
  • ✅ Admin Univ Usulan bisa return ke Pegawai
  • ✅ Status berubah ke "Perbaikan Usulan"
  • ✅ Catatan perbaikan tersimpan

Ringkasan Perbaikan

Alur yang Diperbaiki:

  1. Pegawai → Admin Fakultas (validasi awal)
  2. Admin Fakultas → Admin Univ Usulan (jika valid)
  3. Admin Univ Usulan → Admin Fakultas/Pegawai (jika perlu perbaikan)
  4. Tombol "Dokumen Dikirim ke Universitas" hanya muncul setelah "Usulkan ke Universitas"

Kondisi Tampilan Dokumen:

  • Status "Diajukan": Dokumen TIDAK ditampilkan
  • Status "Perbaikan Usulan": Dokumen DITAMPILKAN dan BISA DEDIT
  • Status "Diusulkan ke Universitas": Dokumen DITAMPILKAN tapi TIDAK BISA DEDIT
